Customer Support Triage Demo
This project demonstrates using multiple GenAI agents to automate the triage of customer support tickets. The system is built with a CLI that deploys, manages, and exposes each agent as an independent API endpoint.

ğŸ§© Project Overview
Goal:
Automatically summarize, analyze sentiment, and route support tickets to the appropriate queue using GenAI agents hosted as microservices/APIs.

Frontend:
A simple dashboard that allows you to upload/view tickets, shows pipeline stages for each ticket, the output at each agent stage, and route/escalation status.

Backend Agents:

/summarize: Ticket Summarizer Agent
Summarizes raw customer messages for conciseness and clarity.

/sentiment: Sentiment Analyzer Agent
Detects urgency, anger, or negative sentiment in tickets.

/route: Priority Router Agent
Decides whether a ticket needs escalation to human support based on summary and sentiment.
